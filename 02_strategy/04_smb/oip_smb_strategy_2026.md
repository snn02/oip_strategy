# СМБ-направление ОИП (Service Factory)

**Ответственный:** Дмитрий  
**Горизонт планирования:** 2026 год

Документ описывает стратегию, цели, OKR и KPI СМБ-направления
Отдела интернет-проектов (ОИП).

Документ служит управленческой рамкой для руководителя направления
и помогает ответить на три ключевых вопроса:

1. Зачем существует СМБ-направление.
2. Какие изменения должны произойти в 2026 году.
3. По каким показателям оценивается результат.

СМБ-направление — **быстрый и прибыльный конвейер небольших задач ОИП**,
работающий по модели Service Factory.

Руководитель направления управляет:

- потоком входящих обращений
- скоростью обработки запросов и подготовки оценок
- экономикой задач
- развитием клиентского портфеля
- передачей клиентов в сопровождение при росте регулярного объёма

---------------------------------------------------------------------

## Связь с другими направлениями ОИП

| Направление | Роль | Интерфейс с СМБ-направлением |
|---|---|---|
| Продажи (Елена) | Поставляет входящий поток обращений | Дмитрий принимает лиды и готовит оценки |
| Производство (Сербин) | Выполняет СМБ-задачи | 1 разработчик + аналитик Б24 (30%) закреплены приоритетно; Дмитрий ставит задачи и расставляет приоритеты |
| Проектное (Виктория) | Принимает крупные задачи | Граница: задачи свыше 80 часов передаются в проектное направление |
| Сопровождение (Ирина) | Принимает клиентов с регулярным потоком задач | Граница: клиенты верхнего СМБ и КОРП с регулярным объёмом задач передаются Ирине; нижний СМБ остаётся у Дмитрия |

---------------------------------------------------------------------

# 1. Стратегические цели направления

Цели отвечают на вопрос **«зачем существует СМБ-направление»**
и задают контекст для OKR и KPI.

---------------------------------------------------------------------

## Цель 1. СМБ-направление — стабильный источник валовой прибыли ОИП

СМБ-задачи обеспечивают регулярный поток валовой прибыли
и быстрый денежный поток для ОИП.

Это достигается через:

- быстрый отклик на обращения клиентов
- быстрые оценки задач
- короткий цикл выполнения работ
- повторные продажи существующим клиентам
- контроль экономики задач

Задача считается успешной, если:

- результат принят заказчиком
- задача выполнена в согласованный срок
- задача принесла прибыль
- клиент готов продолжать работу

---------------------------------------------------------------------

## Цель 2. СМБ-направление работает как управляемый конвейер задач

СМБ-направление работает по модели **Service Factory**:

- быстрый приём входящих обращений
- быстрая подготовка оценки
- выполнение задач короткими итерациями
- прозрачная очередь задач
- отсутствие управленческого хаоса
- библиотека переиспользуемых решений для типовых СМБ-задач

**Граница задач направления:**

- задачи **до 80 часов** выполняются в СМБ-направлении
- задачи **более 80 часов** передаются в проектное направление

Это позволяет сохранять потоковую модель работы без накопления крупных
неуправляемых задач внутри направления.

---------------------------------------------------------------------

## Цель 3. СМБ-направление развивает клиентский портфель

Задача направления — сформировать пул клиентов,
которые регулярно заказывают небольшие доработки и развитие решений.

Повторные продажи становятся основным источником задач.

Дополнительно — выявление клиентов верхнего СМБ и КОРП с растущим объёмом задач
и своевременная передача их в сопровождение.

---------------------------------------------------------------------

## Цель 4. СМБ-направление помогает стабилизировать загрузку производства

СМБ-задачи сглаживают загрузку команды между проектами:

- короткие задачи запускаются быстро
- гибкое распределение между специалистами
- СМБ работает как **механизм выравнивания загрузки производства**

---------------------------------------------------------------------

# 2. OKR СМБ-направления

OKR фиксируют изменения системы управления СМБ-задачами,
которые должны произойти в 2026 году.

OKR — это не план задач, а описание **нового состояния системы**.

---------------------------------------------------------------------

# Objective 1

## СМБ-направление приносит плановую валовую прибыль

СМБ управляет не только задачами, но и экономикой небольших работ.

### KR1 — Прибыльность задач

Не менее **90% задач завершаются с положительной маржой**.

---------------------------------------------------------------------

### KR2 — Доля повторных продаж

Не менее **60% задач выполняется для существующих клиентов**.

---------------------------------------------------------------------

# Objective 2

## СМБ-направление работает как быстрый сервис для клиентов

Главная ценность направления — скорость реакции.

### KR1 — Скорость подготовки оценки

Среднее время от обращения клиента до отправки оценки
не превышает **1 рабочего дня**.

---------------------------------------------------------------------

### KR2 — Скорость выполнения задач

Средний цикл выполнения СМБ-задачи
не превышает **5 рабочих дней**.

---------------------------------------------------------------------

### KR3 — Соблюдение сроков задач

Не менее **90% задач выполняются в согласованный срок**.

---------------------------------------------------------------------

# Objective 3

## СМБ-направление формирует устойчивый поток задач

Задача — обеспечить постоянный поток небольших работ.

### KR1 — Поток входящих обращений

СМБ-направление обрабатывает **30–50 входящих обращений в месяц**.

---------------------------------------------------------------------

### KR2 — Подготовка предложений

Сформировано **20–30 предложений в месяц**.

---------------------------------------------------------------------

### KR3 — Выполненные задачи

Выполняется **10–20 задач в месяц**.

---------------------------------------------------------------------

# Objective 4

## СМБ-направление масштабируется без управленческого хаоса

Рост задач не должен увеличивать управленческую сложность.

### KR1 — Прозрачность очереди задач

Существует единая очередь задач, в которой в любой момент видно:

- какие задачи приняты
- какие задачи оцениваются
- какие задачи выполняются
- какие задачи завершены

---------------------------------------------------------------------

### KR2 — Стандартизация задач

Сформирован каталог типовых задач и типовых оценок.

---------------------------------------------------------------------

### KR3 — Библиотека переиспользуемых решений

Сформирована библиотека типовых решений для наиболее частых типов СМБ-задач.

Библиотека считается сформированной, если покрывает
**не менее 5 типов задач** и включает для каждого:

- описание типового решения
- ссылку на реализованный пример
- типовую оценку трудозатрат

Библиотека снижает время оценки и выполнения повторяющихся задач
и повышает предсказуемость экономики направления.

---------------------------------------------------------------------

# 3. KPI руководителя СМБ-направления

KPI оцениваются **ежеквартально** и влияют на расчёт премии.

| KPI | Показатель | Формула / Метрика | Цель |
|---|---|---|---|
| KPI 1 | Валовая прибыль СМБ-направления | Выручка – прямая себестоимость | ≥ 80% плана (порог премии) |
| KPI 2 | Количество выполненных задач | Выполненные задачи / период | ≥ 10 задач в месяц |
| KPI 3 | Доля повторных продаж | Задачи от существующих клиентов / все задачи | ≥ 60% |
| KPI 4 | Скорость подготовки оценки | Среднее время от лида до оценки | ≤ 1 рабочий день |
| KPI 5 | Средняя маржинальность задач | Валовая прибыль / выручка задач | Устойчивая положительная маржа |

Подробное описание KPI в [oip_smb_metrics_guide.md](./oip_smb_metrics_guide.md).

---------------------------------------------------------------------

## KPI 1 — Валовая прибыль СМБ-направления

Ключевой показатель эффективности направления.
Отражает итоговую экономику портфеля небольших задач за квартал.

Порог включения премии:

**ВП ≥ 80% от плана**

---------------------------------------------------------------------

## KPI 2 — Количество выполненных задач

Отражает пропускную способность конвейера.

Цель: **≥ 10 задач в месяц**

---------------------------------------------------------------------

## KPI 3 — Доля повторных продаж

Отражает лояльность клиентского портфеля и устойчивость потока задач.

Цель: **≥ 60%**

---------------------------------------------------------------------

## KPI 4 — Скорость подготовки оценки

Отражает скорость реакции направления — ключевое конкурентное преимущество СМБ.

Цель: **≤ 1 рабочий день**

---------------------------------------------------------------------

## KPI 5 — Средняя маржинальность задач

Отражает качество экономики задач.

Цель: поддержание устойчивой положительной маржинальности задач.

---------------------------------------------------------------------

# 4. Управленческая логика направления

## Зона ответственности Дмитрия

Руководитель СМБ-направления отвечает за:

- приём входящих обращений и подготовку оценок
- постановку задач в производство (Сербин) и расстановку приоритетов
- соблюдение сроков и качества выполнения задач
- экономику задач: маржинальность и валовую прибыль направления
- развитие клиентского портфеля и повторные продажи
- соблюдение границ направления: передачу крупных задач Виктории и клиентов верхнего СМБ/КОРП — Ирине

Руководитель **не отвечает** за:

- техническую реализацию задач (зона Сербина)
- коммерческие переговоры о новых клиентах (зона Елены)
- задачи крупнее 80 часов (зона Виктории)
- сопровождение клиентов верхнего СМБ и КОРП (зона Ирины)

---------------------------------------------------------------------

## Граница передачи клиентов в сопровождение

Клиент передаётся из СМБ в сопровождение (Ирина), если:

- клиент относится к сегменту **верхнего СМБ или КОРП**
- у клиента формируется **регулярный поток задач**

Клиенты **нижнего СМБ** с регулярными задачами остаются у Дмитрия.

---------------------------------------------------------------------

## Цепочка управления конвейером

```
Входящее обращение
        ↓
Быстрая оценка (≤ 1 день)
        ↓
Выполнение (цикл ≤ 5 дней)
        ↓
Принятый результат (≥ 90% в срок)
        ↓
Повторная продажа (≥ 60% задач — существующие клиенты)
        ↓
Валовая прибыль направления
```

**Если ломается скорость оценки — тормозится весь конвейер.**

Диагностика при падении валовой прибыли:

- ВП падает → смотрим маржинальность задач и количество выполненных задач
- Маржинальность низкая → смотрим перерасход оценок (интерфейс с Сербиным)
- Задач мало → смотрим входящий поток и скорость оценки
- Поток есть, задачи не запускаются → смотрим очередь и загрузку производства

---------------------------------------------------------------------

# 5. Комитет СМБ-направления

Комитет проводится **раз в 2 недели**.

Формат — быстрый обзор состояния конвейера.

| Блок | Показатель | Норма / Интерпретация |
|---|---|---|
| Экономика | Валовая прибыль направления | ≥ 80% плана |
| Экономика | Средняя маржинальность задач | Устойчивая положительная |
| Поток | Входящие обращения | 30–50 в месяц |
| Поток | Выполненные задачи | ≥ 10 в месяц |
| Скорость | Среднее время оценки | ≤ 1 рабочий день |
| Скорость | Средний цикл выполнения | ≤ 5 рабочих дней |
| Скорость | Соблюдение сроков | ≥ 90% задач |
| Портфель | Доля повторных продаж | ≥ 60% |
| Портфель | Передано клиентов в сопровождение | Тренд / факт за период |
| Масштабирование | Статус библиотеки решений | По плану / отставание |

**Цветовая модель здоровья направления:**

- 🟢 **Зелёный** — все показатели в норме; конвейер работает
- 🟡 **Жёлтый** — один показатель вне нормы; требует внимания
- 🔴 **Красный** — два и более показателя вне нормы; нужны управленческие действия
